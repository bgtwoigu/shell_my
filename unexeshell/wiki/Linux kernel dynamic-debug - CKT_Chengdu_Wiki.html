[编辑] 使用方式

内核中提供了一种新的输出log的机制dynamic-debug 简单的说就是：在你需要的时候输出log，不需要的时候可以关闭。这样既避免了大量的log影响性能（严重的时候可能是kernel挂掉），又能在需要的时候输出log。
使用也能简单：
首先它是针对 pr_debug()/dev_dbg() print_hex_dump_debug()/print_hex_dump_bytes()这4个函数输出的log.
1、当然你首先需要的是enable 这个CONFIG_DYNAMIC_DEBUG配置
2、然后启动的是把debugfs mount到摸个目录。 然后在你就能看见这样的一个文件：<debugfs>/dynamic_debug/control 对于我们收集到目录是/d/dynamic_debug/control 当你cat /d/dynamic_debug/control的时候就会出现： 

drivers/rtc/qpnp-rtc.c:654 [qpnp_rtc]qpnp_rtc_shutdown =p "Disabling alarm interrupts\012"
drivers/rtc/qpnp-rtc.c:115 [qpnp_rtc]qpnp_rtc_set_time =p "Seconds value to be written to RTC = %lu\012"
drivers/rtc/qpnp-rtc.c:376 [qpnp_rtc]qpnp_rtc_read_alarm =p "Alarm set for - h:r:s=%d:%d:%d, d/m/y=%d/%d/%d\012"
drivers/rtc/qpnp-rtc.c:279 [qpnp_rtc]qpnp_rtc_read_time =p "secs = %lu, h:m:s == %d:%d:%d, d/m/y = %d/%d/%d\012"
drivers/rtc/qpnp-rtc.c:342 [qpnp_rtc]qpnp_rtc_set_alarm =p "Alarm Set for h:r:s=%d:%d:%d, d/m/y=%d/%d/%d\012"
drivers/rtc/qpnp-rtc.c:609 [qpnp_rtc]qpnp_rtc_probe =p "Probe success !!\012"

[编辑] 让它输出log：

echo 'file qpnp-rtc.c line 115 +p' > /d/dynamic_debug/control

这样就使能了qpnp-rtc.c这个文件的115行的打印了。

[编辑] 不让它输出log：

echo 'file qpnp-rtc.c line 115 -p' > /d/dynamic_debug/control

这样就禁止了qpnp-rtc.c这个文件的115行的打印了。

[编辑] 打开摸个文件的log

echo 'file qpnp-rtc.c +p' > /d/dynamic_debug/control

[编辑] 关闭摸个文件log

echo 'file qpnp-rtc.c -p' > /d/dynamic_debug/control

[编辑] 打印tid、函数名、模块名

线程id：echo 'file qpnp-rtc.c -pt' > /d/dynamic_debug/control
模块名：echo 'file qpnp-rtc.c -pm' > /d/dynamic_debug/control
函数名：echo 'file qpnp-rtc.c -pf' > /d/dynamic_debug/control

[编辑] pr_debug机制分析

[编辑] pr_debug()函数原型

函数定义： 

#if defined(CONFIG_DYNAMIC_DEBUG)
/* dynamic_pr_debug() uses pr_fmt() internally so we don't need it here */
#define pr_debug(fmt, ...) \
	dynamic_pr_debug(fmt, ##__VA_ARGS__)
#elif defined(DEBUG)
#define pr_debug(fmt, ...) \
	printk(KERN_DEBUG pr_fmt(fmt), ##__VA_ARGS__)
#else
#define pr_debug(fmt, ...) \
	no_printk(KERN_DEBUG pr_fmt(fmt), ##__VA_ARGS__)
#endif

因为我们是定义了CONFIG_DYNAMIC_DEBUG所以它展开时是dynamic_pr_debug(fmt, ##__VA_ARGS__)

dynamic_pr_debug展开： 

#define DEFINE_DYNAMIC_DEBUG_METADATA(name, fmt)		\
	static struct _ddebug  __aligned(8)			\
	__attribute__((section("__verbose"))) name = {		\
		.modname = KBUILD_MODNAME,			\
		.function = __func__,				\
		.filename = __FILE__,				\
		.format = (fmt),				\
		.lineno = __LINE__,				\
		.flags =  _DPRINTK_FLAGS_DEFAULT,		\
	}
 
#define dynamic_pr_debug(fmt, ...)				\
do {								\
	DEFINE_DYNAMIC_DEBUG_METADATA(descriptor, fmt);		\
	if (unlikely(descriptor.flags & _DPRINTK_FLAGS_PRINT))	\
		__dynamic_pr_debug(&descriptor, pr_fmt(fmt),	\
				   ##__VA_ARGS__);		\
} while (0)

从上面可知：
1、它初始了struct _ddebug类型的变量，这个结构体，的各个字段分别放的是 函数名 文件名 打印格式 所在行号 和 默认flag，并且它将所有的这个结构体放在"__verbose"这个段中。
2、它将这个变量传给了__dynamic_pr_debug这个函数。
__dynamic_pr_debug函数： 

int __dynamic_pr_debug(struct _ddebug *descriptor, const char *fmt, ...)
{
	va_list args;
	int res;
	struct va_format vaf;
	char buf[PREFIX_SIZE];
 
	BUG_ON(!descriptor);
	BUG_ON(!fmt);
 
	va_start(args, fmt);
 
	vaf.fmt = fmt;
	vaf.va = &args;
 
	res = printk(KERN_DEBUG "%s%pV",
		     dynamic_emit_prefix(descriptor, buf), &vaf);
 
	va_end(args);
 
	return res;
}
EXPORT_SYMBOL(__dynamic_pr_debug);

这个函数最终调用的是printk()打印
