[编辑] 第一章：what + Application of scene

[编辑] A: brief 

  ADB sideload is a new feature that was added to AOSP recovery in Jelly Bean.  As of version 2.3, TWRP now supports ADB sideload mode.

ADB sideload is a different ADB mode that you can use to push and install a zip using one command from your computer.

Most likely ADB sideload won't be very useful for your average recovery user, but ADB sideload can be a huge time-saver for a ROM developer

[编辑] B:此种刷机方式适合的场景

1：由于fastboot client实现是在lk或者在uboot中，所以这种刷机只适用于spl + lk/uboot完整的情况下

2：分区大小和分区数量没有发生变化的情况下<<<除非先更新lk/uboot，并且earse all block device(Nand/emmc)

3：此中刷机方式，默认不会更新spl + lk/uboot;当然修改代码可以实现（修改办法不在此wiki范围内，私聊）

3：适合用ubuntu环境，因为windows的命名不能补齐，键入命令略显不爽，windows的还不如用CPU厂商给的刷机工具;

[编辑] 第二章：文件准备

adb sideload 刷机的方式需要的文件：recovery + full_ota_zip(xxxx.zip);

下面针对不同方案给出编译出这些文件的命令步骤

[编辑] A:标准AOSP准备文件步骤

$$$$. build/envsetup.sh

$$$$lunch you_project

$$$$make otapackage -jxx

[编辑] B:qcom准备文件步骤

$$$$.build/envsetup.sh

$$$$lunch you_project

$$$$make -jxx

[编辑] C:MTK准备文件步骤

$$$$./mk -t $you_project n otapackage

[编辑] 第三章：刷机命令

$$$$adb reboot-bootloader

$$$$fastboot boot out/target/product/ckt95_lw_kk/recovery.img

downloading 'boot.img'...

OKAY [ 0.389s]

booting...

OKAY [ 0.052s]

finished. total time: 0.440s

$$$$手机端apply update from ADB

$$$$PC端输入adb devices可看到如下信息：

zhl@zhl-OptiPlex-9020:~/mycode/2hicam/mt6595_kk$ adb devices

List of devices attached

CKT HICAM sideload

$$$$键入命令

zhl@zhl-OptiPlex-9020:~/mycode/2hicam/mt6595_kk$ adb sideload out/target/product/ckt95_lw_kk/ckt95_lw_kk-ota-6d0b4bf.zip

sending: 'sideload' 100%

手机端会自动完成升级。

[编辑] 第四章：结束语

有人肯定会问，为什么要用fastboot boot out/target/product/ckt95_lw_kk/recovery.img命令加载外置的recovery而不是用手机flash中的recovery。

因为recovery 升级的时候最开始的阶段会去检测签名是不是update中一样，不一致的话会不让升级，这样做是为了刷错包。

当然你也可以修改recovry代码来屏蔽签名验证
