Kernel option for ramdump

CONFIG_MSM_DLOAD_MODE

Get ramdump information

After kernel panic occur in qualcomm phone, The phone will go into ramdump mode automatically. If you want to get the ramdump information form the phone, you can follow these steps. 

 1.Put the usb cable between the phone and computer, The port 9006 will display in the device management.
   

 2.Open memory debug app in QPST tool.
   

 3.Click browse and watch dump process, after finish the dump, the phone will reboot automatically.
   

 4.Right click the earth logo and choose open the data directory.
    

 5.Open the sahara dictory , the ramdump file is in the Port_COMX dictory. The Port_COMX is the port in your device management.

Parse the ramdump information

Parse the ramdump information in ubuntu , do as follow 

 1.Download the tools
   git clone git://www.codeaurora.org/platform/vendor/qcom-opensource/tools.git

 2.Make a new file named local_settings.py in the tool directory

note:如果是64位的内核需要配置如下信息： gdb64_path，nm64_path，objdump64_path 

example:
gdb64_path = "/home/mask/work/msm8939-la-2-1/LINUX/android/prebuilts/gcc/linux-x86/aarch64/aarch64-linux-android-4.9/bin/aarch64-linux-android-gdb"
nm64_path = "/home/mask/work/msm8939-la-2-1/LINUX/android/prebuilts/gcc/linux-x86/aarch64/aarch64-linux-android-4.9/bin/aarch64-linux-android-objdump"
objdump64_path = "/home/mask/work/msm8939-la-2-1/LINUX/android/prebuilts/gcc/linux-x86/aarch64/aarch64-linux-android-4.9/bin/aarch64-linux-android-objdump"



 3.Run the cmd  python ramparse.py -a dump_1/crush_ramdump -v dump_1/vmlinux/vmlinux -o out/ -x
      -a AUTODUMP, --auto-dump=AUTODUMP
                       Auto find ram dumps from the path
      -v VMLINUX, --vmlinux=VMLINUX
                       vmlinux path
      -o OUTDIR, --outdir=OUTDIR
                       Output directory
      -x, --everything      Output everything 
    
    
note:如果是64位需要加上如下参数：--64-bit

 4.The output information in the out directory
