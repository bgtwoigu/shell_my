cpufreq core[编辑 | 编辑源代码]

cpufreq core主要完成一下工作：

提供两个api，cpufreq_register_driver和cpufreq_register_governor;

管理一个cpufreq_policy　update　work;创建文件节点,供调试和用户空间设置governor和其他参数;

向内核注册pm　callback

cpufreq core实现code: kernel/drivers/cpufreq/cpufreq.c

cpufreq_register_driver[编辑 | 编辑源代码]

用于实现平台化差异参数cpufreq_driver，常见实例化代码：vendor-cpufreq.c 比如qcom-cpufreq.c，

最终的目地是把对应的governor中的cpu频率和vcck电压设置到对应的值，

比如设置PLL分频，和调用PMU输出对应的电压等，vendor-cpufreq.c通过cpufreq_register_driver注册到cpufreq core,供cpufreq core调用

static int __init msm_cpufreq_register(void)
{
    int cpu, rc;
 
    for_each_possible_cpu(cpu) {
        mutex_init(&(per_cpu(cpufreq_suspend, cpu).suspend_mutex));
        per_cpu(cpufreq_suspend, cpu).device_suspended = 0;
    }
 
    rc = platform_driver_probe(&msm_cpufreq_plat_driver,
                   msm_cpufreq_probe);
    if (rc < 0) {
        /* Unblock hotplug if msm-cpufreq probe fails */
        unregister_hotcpu_notifier(&msm_cpufreq_cpu_notifier);
        for_each_possible_cpu(cpu)
            mutex_destroy(&(per_cpu(cpufreq_suspend, cpu).
                    suspend_mutex));
        return rc;
    }
 
    register_pm_notifier(&msm_cpufreq_pm_notifier);
    return cpufreq_register_driver(&msm_cpufreq_driver);
}


cpufreq_register_governor[编辑 | 编辑源代码]

此API用于向cpufreq注册一个governor,　实现调频策略的差异化，

一个实例化cpu一般可以绑定多种governor，具体激活哪个由用户空间决策（当然

还可以让只绑定一个默认的governor,缺点是更改governor需要重新修改代码，

且不便于动态调整governor,对应的宏开关CONFIG_CPU_FREQ_DEFAULT_GOV_CONSERVATIVE）

struct cpufreq_governor cpufreq_gov_conservative = {
    .name           = "conservative",
    .governor       = cs_cpufreq_governor_dbs,
    .max_transition_latency = TRANSITION_LATENCY_LIMIT,
    .owner          = THIS_MODULE,
};
 
static int __init cpufreq_gov_dbs_init(void)
{
    return cpufreq_register_governor(&cpufreq_gov_conservative);
}
 
static void __exit cpufreq_gov_dbs_exit(void)
{
    cpufreq_unregister_governor(&cpufreq_gov_conservative);
}

misc[编辑 | 编辑源代码]

此外cpufreq　core还维护一个handle_update，用来实现cpufreq　core最重要的功能：更新policy;

创建文件节点,调用路径cpufreq_add_dev－－－__cpufreq_add_dev－－－cpufreq_add_policy_cpu－－－－sysfs_create_link,供调试和用户空间设置governor和其他参数;

向内核注册pm　callback,调用路径register_syscore_ops
