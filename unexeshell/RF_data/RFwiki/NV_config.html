RF NV

  • #1878(NV_RF_HW_CONFIG_I)：89
该项用于配置RFCard，其值为对应的RF card ID。在RF task启动时会加载对应的RF card，以便初始化相应的RF硬件设备。
所有已定义的RFCard ID可以参考:modem_proc/rfc_dimepm/api/rfc_hwid.h
  • #1877(NV_RF_BC_CONFIG_I)：82313600
该项用于配置RF主通路上所支持的频段(包括GSM、WCDMA及CDMA)。其中每一个bit表示一个频段，“1”则表示支持该频段。
而“82313600”表示支持GSM 850/900/1800/1900以及WCDMA Band1/2/5。
关于每一个bit位的描述请参考modem_proc/mmcp/api/public/sys.h中的sys_sband_e_type。
  • #6828(NV_LTE_BC_CONFIG_I)：65609
该项用于配置LTE在主通路上所支持的频段。其中每一个bit表示一个频段，“1”则表示支持该频段。
而“65609”表示支持LTE Band1/4/7/17。
关于每一个bit位的描述请参考modem_proc/mmcp/api/public/sys.h中的sys_sband_lte_e_type。
  • #4548(NV_RF_BC_CONFIG_DIV_I)：0
该项用于配置RF分集通路(DRX)上所支持的频段。默认是支持LTE分集的，所以LTE不需要配置，而其它频段需要根据具体的硬件设计进行配置。
在A和Spirit项目中只有LTE具有分集通路，所以该项为0。
  • #22605(NV_TDSCDMA_BC_CONFIG_I)：0
  • #24203(NV_TDSCDMA_BC_CONFIG_DIV_I)：0
以上两项用于配置TDSCDMA在主通路和分集通路(DRX)上所支持的频段。
  • #22131(NV_RF_BC_CONFIG_C2_I)：0
  • #23387(NV_RF_BC_CONFIG_C3_I)：0
以上两项用于配置Rx path 2和Rx path3通路所支持的频段。由于这种情况一般需要2个甚至3个Transceiver，所以A和Spirit上并未使用到。
  • #2029(NV_WCDMA_PA_RANGE_MAP_I)：3,2,2,2
  • #538(NV_WCDMA_R1_FALL_I)：223
  • #537(NV_WCDMA_R1_RISE_I)：253
  • #2018(NV_WCDMA_R2_FALL_I)：32767
  • #2017(NV_WCDMA_R2_RISE_I)：32767
  • #2020(NV_WCDMA_R3_FALL_I)：32767
  • #2019(NV_WCDMA_R3_RISE_I)：32767
NV_WCDMA_PA_RANGE_MAP_I用于配置WCDMA Band1的PA state[0:3]的增益模式，其中Range3对应PA的低功率模式，Range2表示高功率模式。
而NV_WCDMA_R[X]_FALL_I和NV_WCDMA_R[X]_RISE_I用于配置WCDMA Band1对应PA的切换点，即当发射功率达到某个切换点时，会跳转到另一个PA state。如下图所示：

另外R2和R3中的值为“32767”，表示PA state2和 state3不可用，PA工作时不会切换到这两个state，只会在state0和state1之间进行切换。
WCDMA的其它频段配置与此WCDMA Band1一样。
注意：请务必确保PA State与PA切换点配置正确，否则会导致信令测试时最大功率输出异常。
  • #6717(NV_LTE_B1_PA_RANGE_MAP_I)：2,2,2,3
  • #6719(NV_LTE_B1_PA_RISE_FALL_THRESHOLD_I)：1023,1023,1023,1023,830,800
NV_LTE_B1_PA_RANGE_MAP_I用于配置LTE Band1的PA state[0:3]的增益模式，其中Range2对应PA的高功率模式，Range3表示低功率模式。
而NV_LTE_B1_PA_RISE_FALL_THRESHOLD_I用于配置LTE Band1对应PA的切换点，即当发射功率达到某个切换点时，会跳转到另一个PA state。如下图所示：

如果切换点的值为“1023”，则表示对应的PA state不可用。即LTE Band1的PA state2和 state3不可用，PA工作时不会切换到这两个state，只会在state2和state3之间进行切换。
LTE其它频段的配置与LTE Band1类似。但对于TDD-LTE而言，通常PA Range只有0和1，而PA state只有0,1,2。
注意：请务必确保PA State与PA切换点配置正确，否则会导致信令测试时最大功率输出异常。

Protocol NV

  • #10(NV_PREF_MODE_I)：SUB0:31, SUB1:13
此NV项用于配置首选网络模式，“31”表示GWL，"13"表示GSM only，而具体的搜网顺序由Modem配置的RAT order list决定。

该项用于配置SIM卡的CS及PS业务。其中“2”表示支持CS+PS，“0”表示CS only，而PS由默认的移动数据卡决定。另外如果要使用LTE，则需要支持PS业务。

此项用于配置3GPP协议中WCDMA RRC的版本。如果要使UE支持HSPA+，则必须将其配置为Rel7以上的版本。详细的说明如下：
  • #4118(NV_HSDPA_CAT_I)：inactive
此项用于配置UE支持HSDPA的Category。如果此项未激活，则软件默认设置的是24。另外参照3GPP协议，如果要支持DC-HSPA+，则Category必须要大于或等于21。
具体请参考：http://en.wikipedia.org/wiki/HSDPA
  • #911(NV_MULTISLOT_CLASS_I)：inactive
  • #2509(NV_EDGE_MULTISLOT_CLASS_I)：inactive
以上两项用于分别配置GPRS和EDGE的MulitSlot class。如果此两项未激活，则软件默认设置的是33。
注意：在Telcel的CMUgo自动化测试中，由于multi Slot配置不正确，导致GPRS无法成功建立连接，此时则需要将以上两项设置成12，即配置成GPRS MultiSlot Class 12。
  • #849(NV_NET_SEL_MODE_PREF_I )：0
该项用于配置UE的网络选择模式，默认设置是AUTOMATIC，即“0”。如果设置成其它的值，可能会出现注册不上网络的情况。所以建议将网络选择模式设置为自动。
另外如果用户在设置中手动将网络选择模式选为手动模式，则该项的值会变为“1”。

Misc

  • #453(NV_FTM_MODE_I)：0
该项用于配置FTM模式，为“1”则表示进入FTM模式。而FTM模式主要用于校准和非信令测试，此时UIM相关功能会被禁用，所以识别不了SIM卡。
  • #947(NV_GPRS_ANITE_GCF_I)：0
该项用于GCF测试。当插入测试卡(MCC为001到012)时，该项会自动被置为“1”，以阻止TCXO关闭，保证GCF测试能够顺利进行。所以在手机待机时会增加大约30mA的电流。
如果要避免此问题，一种方法是将测试卡的MCC除001到012之前的其它值，另一种则是修改代码中mmgsdi_reserved_mcc_tbl[]这个数组，让当前测试卡的MCC不在这个列表中。
  • #74(NV_AUTO_ANSWER_I)：inactive
此项用于设置自动接听功能。一般在自动化测试中需要设置为“1”，即激活自动接听功能，激活后会自动接听所有来电。
  • #550(NV_UE_IMEI_I)：
此项用于记录手机的IMEI，具体格式如下：

  • #4398(NV_UIM_SELECT_DEFAULT_USIM_APP_I)：1
  • #6876(NV_DUAL_STANDBY_CFG_ITEMS_I)：1
  • #6907(NV_UIM_HW_SIM_CONFIG_I)：0
  • #70266(Device Mode)：0
  • #70210(UIM HW Config)：
hw_config.UIM[0].DISABLE_UIM：FALSE
hw_config.UIM[1].DISABLE_UIM：TRUE
  • adb shell setprop persist.radio.multisim.config ssss
以上六项用于将手机配置成SSSS(单卡单待)模式。
  • #4398(NV_UIM_SELECT_DEFAULT_USIM_APP_I)：0
  • #6876(NV_DUAL_STANDBY_CFG_ITEMS_I)：5
  • #6907(NV_UIM_HW_SIM_CONFIG_I)：1
  • #70266(Device Mode)：1/2(DSDS/DSDA)
  • #70210(UIM HW Config)：
hw_config.UIM[0].DISABLE_UIM：FALSE
hw_config.UIM[1].DISABLE_UIM：FALSE
  • adb shell setprop persist.radio.multisim.config dsds/dsda
以上六项用于将手机配置成DSDS(双卡双待)或者DSDA(双卡双通)模式。
注意：如果Modem和AP的多模配置不匹配，可能会导致语音通话无声音。
  • #70210(UIM HW Config)：
此项用于配置SIM相关的硬件Pin脚，包括SIM卡的热插拔功能。
上图展示了SIM1的配置。其中“DISABLE_UIM”用于使能或者关闭当前SIM Slot的功能，而“ENABLE_UIM_HOTSWAP”则用于激活SIM卡的热插拔功能，此时需要连接到UIM的探测引脚。“UIM_HOTSWAP_POLARITY”用于配置SIM卡插入后探测引脚的电平，这里需要参考具体的硬件设计，如果配置错误，会造成系统检测不到SIM卡，即使开机也无法检测到。
另外列表中各个功能引脚对应的GPIO无法修改，即使手动修改后，开机仍会被重置为高通默认的配置。所以硬件设计时最好根据高通的参考设计进行引脚配置。
